# Landed Cost Prediction Model

A machine learning pipeline that predicts total landed cost (INR) for stone slab imports at Purchase Order time — using only pre-arrival features like dimensions, vendor quoted price, and shipment route.

---

## What it does

- Predicts **total landed cost per slab** before customs/freight charges are known
- Flags **cost anomalies** when actual charges deviate significantly from predictions
- Serves predictions via a **REST API** for ERP integration
- Uses **GroupKFold CV** (grouped by shipment) to prevent data leakage

Expected accuracy: R² 0.75–0.90 · MAPE 8–15%

---

## Stack

- **Model** — `GradientBoostingRegressor` (sklearn)
- **API** — FastAPI + Uvicorn
- **Database** — PostgreSQL via SQLAlchemy
- **Serialisation** — joblib

## Usage

**Train:**
```bash
python train.py
python train.py --schema rrmstock --anomaly-percentile 90 --model-path models/model.pkl
```

**Cross-validate only:**
```bash
python cross_validate_model.py
python cross_validate_model.py --n-splits 5
```

**Serve:**
```bash
uvicorn predict_api:app --reload --port 8000
```

## API endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/health` | Model status + CV metrics |
| `GET` | `/features` | Feature list for ERP integration |
| `POST` | `/predict` | Predict for a single slab |
| `POST` | `/predict/batch` | Predict for up to 500 slabs |
| `POST` | `/score/from-db` | Score full inventory, return anomalies |

**Minimal predict request:**
```json
{
  "height": 180, "width": 90, "length": 2, "weight": 1.8,
  "purchase_amount_per_uom_in_vendor_currency": 420,
  "vendor_name": "MI STONE", "port_of_shipment": "MERSIN"
}
```

Supply `total_amount` in the request to trigger anomaly detection against the actual cost.

## Features

All features are available at PO creation time. Post-arrival columns (`customs_duty`, `clearing_charges`, `ocean_fright`, etc.) are intentionally excluded.

| Type | Features |
|---|---|
| Numeric | `height`, `width`, `length`, `weight`, `cft`, `purchase_amount_per_uom_in_vendor_currency`, + engineered: `volume`, `density`, `area`, `weight_x_vendor_price`, `price_per_cft`, `purchase_month`, `purchase_quarter` |
| Categorical | `product_name`, `color`, `country`, `vendor_name`, `broker_name`, `vendor_currency`, `port_of_shipment`, `country_of_shipment`, `port_of_discharge` |

---

## Database schema

Expects a PostgreSQL schema (default `rrmstock`) with:

| Table | Role |
|---|---|
| `raw_material_inventory` | One row per slab — dimensions, product, cost columns |
| `material_receive_note` | Shipment-level — vendor, broker, port, currency |
| `purchase_order` | PO-level — currency confirmation |

Join path: `rmi.mrn_id → mrn.id → mrn.po_number → po.purchase_order_number`
